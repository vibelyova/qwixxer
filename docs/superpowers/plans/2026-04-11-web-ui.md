# Web UI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a web-based Qwixx game where humans play against our bots in the browser, with all game logic running client-side via WASM.

**Architecture:** Rust game engine compiled to WASM via wasm-bindgen, exposing a `WebGame` API. TypeScript frontend renders a colorful Qwixx scoresheet and handles user interaction. No backend — everything runs in the browser, hostable on GitHub Pages.

**Tech Stack:** Rust + wasm-bindgen + wasm-pack (WASM), TypeScript + Vite (frontend), HTML/CSS (UI)

---

- [ ] **Step 2: Create `web/src/lib.rs` with WebGame API**

```rust
use wasm_bindgen::prelude::*;
use qwixxer::state::{State, Move, Mark};
use qwixxer::game::DiceSource;
use qwixxer::strategy::Strategy;
use qwixxer::bot;
use rand::rngs::SmallRng;
use rand::{Rng, SeedableRng};
use serde::Serialize;
use std::sync::Arc;

#[wasm_bindgen]
pub struct WebGame {
    player_state: State,
    bot_state: State,
    bot: Box<dyn Strategy>,
    rng: SmallRng,
    active_player: usize, // 0 = bot, 1 = human
    dice: [u8; 6],
    game_over: bool,
}

#[derive(Serialize)]
pub struct GameView {
    player: StateView,
    bot: StateView,
    dice: Vec<u8>,
    white_sum: u8,
    phase: String, // "bot_turn", "your_passive", "your_turn", "bot_passive", "game_over"
    available_moves: Vec<MoveView>,
    game_over: bool,
    winner: String,
}

#[derive(Serialize)]
pub struct StateView {
    rows: Vec<RowView>,
    strikes: u8,
    score: isize,
}

#[derive(Serialize)]
pub struct RowView {
    numbers: Vec<u8>,      // the numbers in order (2-12 or 12-2)
    free: Option<u8>,      // next markable number
    total: u8,             // marks count
    locked: bool,
    ascending: bool,
    color: String,         // "red", "yellow", "green", "blue"
}

#[derive(Serialize)]
pub struct MoveView {
    index: usize,
    description: String,
    is_strike: bool,
}

#[wasm_bindgen]
impl WebGame {
    #[wasm_bindgen(constructor)]
    pub fn new(bot_type: &str) -> WebGame {
        // Create bot based on type
        let bot: Box<dyn Strategy> = match bot_type {
            "ga" => {
                let genes = Arc::new(bot::default_genes());
                let champion = bot::DNA::load_weights_from_bytes(
                    include_bytes!("../../champion.txt"), genes
                );
                Box::new(champion)
            }
            "opportunist" => Box::new(qwixxer::strategy::Opportunist),
            "conservative" => Box::new(qwixxer::strategy::Conservative::default()),
            _ => Box::new(qwixxer::strategy::Opportunist), // fallback
        };

        let mut rng = SmallRng::from_entropy();
        let dice = core::array::from_fn(|_| rng.gen_range(1..=6));

        WebGame {
            player_state: State::default(),
            bot_state: State::default(),
            bot,
            rng,
            active_player: 0,
            dice,
            game_over: false,
        }
    }

    /// Get the current game state as JSON.
    pub fn view(&self) -> String {
        // Build GameView and serialize
        serde_json::to_string(&self.build_view()).unwrap()
    }

    /// Start a new turn: roll dice and process bot's turn if bot is active.
    pub fn next_turn(&mut self) -> String {
        // Roll new dice, process bot turn, return view
        self.dice = core::array::from_fn(|_| self.rng.gen_range(1..=6));
        // ... game logic
        self.view()
    }

    /// Human makes a move (by index from available_moves).
    pub fn make_move(&mut self, move_index: usize) -> String {
        // Apply the chosen move, advance game state
        self.view()
    }

    /// Human skips (on opponent turn).
    pub fn skip(&mut self) -> String {
        // Skip marking on opponent turn
        self.view()
    }
}
```

Note: This is a skeleton. The full game loop logic (turn management, lock propagation, game-over detection) will be fleshed out in Task 3.

- [ ] **Step 1: Add `load_weights_from_bytes` to bot::DNA**

In `src/bot.rs`, add a method that parses champion.txt content from bytes (for embedding in WASM):
```rust
pub fn load_weights_from_bytes(bytes: &[u8], genes: Arc<Vec<GeneFn>>) -> Self {
    let content = std::str::from_utf8(bytes).expect("Invalid UTF-8");
    let weights: Vec<f64> = content
        .lines()
        .map(|line| line.split_whitespace().last().unwrap().parse().unwrap())
        .collect();
    assert_eq!(weights.len(), genes.len(), "Weight count mismatch");
    DNA { weights, genes }
}
```

- [ ] **Step 2: Implement the full game state machine in `web/src/lib.rs`**

The game flow is:
1. Bot's active turn: roll dice → bot makes move → human sees white sum for passive marking
2. Human's passive move: mark or skip on the white sum
3. Human's active turn: roll dice → human picks move from candidates
4. Bot's passive move: bot marks or skips on white sum
5. Propagate locks, check game over, repeat

Each method (`next_turn`, `make_move`, `skip`) advances the state machine and returns the new view.

- [ ] **Step 3: Test by writing a Rust integration test**

```rust
#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn game_completes() {
        let mut game = WebGame::new("opportunist");
        for _ in 0..100 {
            let view: GameView = serde_json::from_str(&game.view()).unwrap();
            if view.game_over { break; }
            // Always pick first move or skip
            if !view.available_moves.is_empty() {
                game.make_move(0);
            } else {
                game.skip();
            }
            game.next_turn();
        }
    }
}
```
